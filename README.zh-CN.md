# 大同语 · Datunwen

[English](README.md) · **简体中文**

**大同语（Datunwen）——一门工程化设计的人造语言：把语法架在音高信道上，用一套「特征化方块文字」书写，力求在汉语级的音节预算内达到拉丁语级的显式度；以零依赖的单文件交互式 HTML 交付：设计书、双语入门课程、三语词典与神经语音示例。**

> 大同语的语法信息由音高信道承载（词类不写进词典，由整词的音高轮廓即时指派），文字是「韩文拼形 × 汉字形声」的二维方块，输入端是纯 ASCII。本仓库包含完整的中文设计书、中/英入门课程、三语词典与神经语音参考音——全部为零依赖的单文件交互式 HTML，浏览器直接打开即用。

## 六十秒读懂这门语言

三条设计公理贯穿一切：

1. **两条信道。** 嘴承载词汇；喉承载语法。每个实词都在一条与词等长的音高轮廓下说出，实时指派它的词性——高平＝论元（`ka`「知识」），下降＝谓语（`kaq`「知道」），上升＝修饰语（`kaz`「认知的」），低平＝助词。拼写上，两个哑的 ASCII 字母承载轮廓：`-q` 降、`-z` 升。每个词无需死记：三条语调规则，而非一部声调词典。
2. **默认零标记。** SVO 语序不花任何成本；格助词只在你偏离默认语序时才出现。时态是一次性设定、带辖域的锚（`tan` 过去 · `nau` 现在 · `fu` 将来），而非逐词的屈折。无时限的陈述合法地不加标记。
3. **一词一屋顶。** 音高轮廓的辖域 ＝ 词 ＝ 每个书写方块上方的屋顶笔画。声音、文字、句法，是同一对象的三个投影。

音系：16 个全球常见辅音（字母 *r*/*l* 拼同一个流音音位）、5 元音 + 4 双元音、唯一韵尾 *-n*——共 288 个可能音节，按纠错码排布（核心词根之间没有危险的最小对立对）。构词：约 250 个语义素折叠成透明的右中心复合词——*su+kau*「自-因」＝自由（causa sui），*su+lin+din*「相连诸自我之值」＝权利。

两句压力测试（在设计书里都可点击发声）：

```
en Xedinga e mau cuq lo mun io wi e tunseitai.
专名 薛定谔 的 猫 位于 在 死 ⊕ 生 的 共存之态
「薛定谔的猫处于死与生的叠加态。」
（io ＝ 这门语言原生的非经典连词：「既-又」，既非 ∧ 亦非 ∨）

mei ren an xinwi sukauq, i lo sudin i sulindin pinq.
∀ 人 于 出生 自因, 且 在 自值 与 相连诸自我之值 齐平
「人人生而自由，在尊严和权利上一律平等。」—— 《世界人权宣言》第一条，
16 个音节：与中文原文等长，且每个量词与谓语都完全显式。
```

## 数据与工具

`lexicon.json` ＝ `{ meta, entries[] }`；每条词条：`f`（词形）· `kind`（root 词根 / pronoun 代词 / particle 助词 / comp 复合）· `cls`（7 个沉默分类符偏旁之一，或 null）· `en` / `zh` 释义 · `srcEN` / `srcZH`（助记来源）· `parts` + `just`（复合词）· `num`（数词标记）· `layer`（**canon** ＝ 出自设计书 · **v0.2-draft** ＝ 机器起草的扩展，已通过机械校验、待实地检验）。要把一条草稿转正，翻转它的 `layer` 并重跑构建器即可。

**音频流水线**（预生成在 `audio/`，构建时不重跑）：Piper 神经 TTS（音色 `id_ID-news_tts-medium`——印尼语恰好几乎 1:1 覆盖大同语的音位表）逐语素合成片段；随后 WORLD 声码器分解每个词、**用语法轮廓整体替换其 F0 轨**（外加声调下倾与助词弱化）再重新合成。对输出做盲重分析，证实语法确实可测地落在声波里：`ka` 平 287→273 Hz，`kaq` 降 318→183 Hz，`kaz` 升 199→314 Hz。页内点击发声的音频，则是另一套完全自足的 Klatt 式共振峰合成器（Web Audio，无需任何素材）。

## 方法与现状

这门语言是在一次 Claude（Anthropic）会话中端到端设计的——音系依据跨语言语速/信息密度数据论证（Coupé, Oh, Dediu & Pellegrino 2019，*Science Advances*），音位表取自类型学频率；词库层由并行的智能体在硬校验器（音节合法性 · 全局唯一性 · 最小对立对防火墙 · 复合词可分解）与人在环仲裁下扩展到 v0.2。已知的取舍在设计书 §6 里诚实记录（未经检验的密度估计、耳语/失歌症的退路、语义分解的哲学负担、特征化文字里的形近风险）。

这里的一切都是**一份设计提案，而非社群语言**——真正有趣的问题是：音高语法与弹性语域这些想法，能否经受住真实学习者的检验。欢迎提 issue、做实验。
